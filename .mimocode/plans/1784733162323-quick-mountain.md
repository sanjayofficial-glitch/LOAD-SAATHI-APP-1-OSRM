# AI Blog Optimization Engine for LoadSaathi

## Overview

You write blog content → system automatically optimizes it for SEO, AEO, and GEO → generates AI images → publishes with full schema markup. One-click optimization from draft to published.

---

## How It Works

```
You write content → Paste into Admin → Click "Optimize" → System auto-generates:
  ✓ Meta title & description (SEO)
  ✓ AI answer block (40-60 words for AI citation)
  ✓ FAQ section extraction
  ✓ Key takeaways for featured snippets
  ✓ Hero image (AI-generated)
  ✓ Schema markup (Article, FAQPage, BreadcrumbList)
  ✓ Target keywords
  → Review → Publish
```

## Step 1: Database Schema

**New migration**: `supabase/migrations/20260722_blog_engine.sql`

```sql
CREATE TABLE blog_posts (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  slug TEXT UNIQUE NOT NULL,
  
  -- Content (you write this)
  title TEXT NOT NULL,
  content TEXT NOT NULL, -- raw markdown or plain text you paste
  category TEXT NOT NULL DEFAULT 'Industry Insights',
  author TEXT NOT NULL DEFAULT 'LoadSaathi Team',
  author_bio TEXT DEFAULT 'The LoadSaathi team builds AI-powered logistics tools for India freight industry.',
  
  -- Auto-generated by optimization
  excerpt TEXT, -- auto-generated summary
  ai_answer_block TEXT, -- 40-60 word snippet for AI citation
  faq JSONB, -- auto-extracted Q&A pairs
  key_takeaways TEXT[], -- bullet points for featured snippets
  hero_image_url TEXT,
  hero_image_prompt TEXT,
  
  -- SEO fields (auto-optimized)
  meta_title TEXT,
  meta_description TEXT,
  meta_keywords TEXT[],
  canonical_url TEXT,
  target_queries TEXT[],
  
  -- Status
  status TEXT DEFAULT 'draft' CHECK (status IN ('draft', 'optimized', 'published', 'archived')),
  published_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  
  -- Schema
  schema_type TEXT DEFAULT 'Article',
  
  -- Analytics
  view_count INTEGER DEFAULT 0
);

CREATE INDEX idx_blog_posts_published ON blog_posts(status, published_at DESC);
CREATE INDEX idx_blog_posts_slug ON blog_posts(slug);
```

## Step 2: Edge Function — `blog-optimize`

**File**: `supabase/functions/blog-optimize/index.ts`

**Input**: Raw blog content (title + body text)
**Output**: Fully optimized blog post data

This function uses Gemini to:

### 2a. SEO Optimization
- Generate meta title (keyword-rich, <60 chars)
- Generate meta description (150-160 chars with primary keyword)
- Extract/generate meta keywords
- Suggest target search queries
- Generate a clean slug from the title

### 2b. AEO Optimization (Answer Engine)
- Generate 40-60 word AI answer block (self-contained definition that AI can cite)
- Extract or create FAQ pairs from the content (4-6 Q&As)
- Generate key takeaways (3-5 bullet points for featured snippets)

### 2c. GEO Optimization (Generative Engine)
- Ensure content has citable statistics with source framing
- Add "According to..." attribution where relevant
- Structure paragraphs to be self-contained (work without surrounding context)
- Include comparison tables if the content compares things

### 2d. Content Enhancement
- Generate excerpt (2-3 sentences, secondary keywords)
- Suggest hero image prompt
- Calculate read time
- Suggest category if not provided

**Gemini prompt structure:**
```
You are an expert SEO + AEO + GEO content optimizer for LoadSaathi, 
India's AI-powered freight marketplace.

Optimize this blog post for:
1. Traditional Google SEO (keywords, meta tags, structure)
2. AI search engines like ChatGPT, Perplexity, Google AI Overviews (extractable answers)
3. AI agents that cite content (self-contained passages, statistics, authority signals)

Return a JSON object with all optimized fields.
```

## Step 3: Edge Function — `blog-image-generate`

**File**: `supabase/functions/blog-image-generate/index.ts`

Uses Pollinations.ai (free, no API key):
```
https://image.pollinations.ai/prompt/{encoded_prompt}?width=1200&height=630&nologo=true
```

The function:
1. Takes an image prompt (auto-generated from blog content)
2. Fetches the image from Pollinations.ai
3. Uploads to Supabase Storage bucket `blog-images`
4. Returns the public URL
