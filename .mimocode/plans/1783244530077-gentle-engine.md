# LoadSaathi Production Hardening Plan

## Overview
LoadSaathi is India's Truck Space Marketplace — a React 19 + Vite + TypeScript SPA with Clerk auth, Supabase DB, and 5 Supabase Edge Functions. The app has critical security holes, code bugs, and UX gaps that must be fixed before launch.

**122 source files** analyzed. **5 edge functions** audited. **10 database tables** reviewed.

---

## Phase 1: Security Fixes (CRITICAL — must ship first)

### 1A. Create shared edge function helper
**Create**: `supabase/functions/_shared/edgeHelpers.ts`

Extract duplicated code from all 5 edge functions into a shared module:
- `getCorsHeaders(req)` — CORS header generation
- `checkRateLimit(ip, max, windowMs)` — rate limiter
- `verifyBearerToken(req)` — Bearer token extraction (not full JWT verify, but structured check)
- `errorResponse(message, status, requestId)` — standardized error responses
- `getRequestIp(req)` — IP extraction from x-forwarded-for

### 1B. Add JWT verification to ALL edge functions
**Files**: All 5 edge functions

Currently:
- `match-ml/index.ts` (line 15-18): Checks Bearer prefix only — does NOT verify JWT signature
- `credit-score/index.ts` (line 36-42): Checks Bearer prefix only
- `credit-insights/index.ts`: Zero auth
- `price-predict/index.ts`: Zero auth
- `gemini-proxy/index.ts`: Zero auth, deployed with `--no-verify-jwt`

**Fix**: Use Supabase's built-in JWT verification. In Deno Edge Functions, import `createClient` from `https://esm.sh/@supabase/supabase-js@2` and use it to verify the token:
```typescript
import { createClient } from "https://esm.sh/@supabase/supabase-js@2";
const supabase = createClient(url, anonKey, { global: { headers: { Authorization: token } } });
const { data: { user }, error } = await supabase.auth.getUser();
if (error || !user) return 401;
```

Apply to: `credit-insights`, `price-predict`, `gemini-proxy`, `match-ml`, `credit-score`

### 1C. Credit score ownership check
**File**: `supabase/functions/credit-score/index.ts`

After JWT verification, add: `if (decodedSub !== userId) return 403`. Currently any authenticated user can query anyone's credit score.

### 1D. Rate limiting on gemini-proxy
**File**: `supabase/functions/gemini-proxy/index.ts`

Add rate limiter (5 req/min per IP) matching the pattern in `credit-insights/index.ts:237-252`.

### 1E. Rate limiting on credit-score
**File**: `supabase/functions/credit-score/index.ts`

Currently has NO rate limiting. Add 10 req/min per IP.

### 1F. Notification RPC ownership validation
**Create**: `supabase/migrations/20260705_fix_notification_security.sql`

The `create_notification` SECURITY DEFINER function accepts any `p_user_id`. Fix: add ownership check inside the function body, or restrict the INSERT RLS policy to only allow self-inserts.

## Phase 2: Bug Fixes

### 2A. Fix duplicate `const authHeader`
**File**: `supabase/functions/credit-score/index.ts`

Line 36: `const authHeader = req.headers.get("Authorization")`
Line 61: `const authHeader = \`Bearer ${supabaseKey}\``

Same variable name declared twice in same scope — runtime error. Fix: rename line 61 to `const serviceAuthHeader` and update references on lines 69, 86.

### 2B. Standardize RLS auth expressions
**File**: `supabase/migrations/20260705_standardize_rls_auth.sql` (new)

The `credit_scores` migration (`20260702_credit_scores.sql`) uses `auth.uid()::text` while the canonical `final-schema.sql` uses `auth.jwt()->>'sub'`. For Clerk integration, `auth.jwt()->>'sub'` is correct. Create a migration to fix the credit_scores RLS policies.

Also: `complete-setup.sql` uses `auth.uid()::text` throughout — rename it to `complete-setup-archive.sql` to prevent confusion.

## Phase 3: Code Cleanup

### 3A. Add LRU eviction to OSRM cache
**File**: `src/utils/osrm.ts`

Add max size (500 entries) with eviction of oldest 50 entries when full. Replace the plain `Map` with an LRU-style cache.

### 3B. Don't cache geocode misses
**File**: `src/utils/geocode.ts`

Line 46: `geocodeCache.set(cacheKey, null)` — caches null for 24h, blocking retries for misspelled cities. Fix: remove this line entirely.

### 3C. Delete empty App.css
**File**: `src/App.css` — contains only `/* LoadSaathi App Styles */`. Confirmed NOT imported anywhere (grep returned 0 matches). Delete the file.

### 3D. Add correlation IDs to edge functions
**File**: All 5 edge functions + `_shared/edgeHelpers.ts`

Generate `crypto.randomUUID().slice(0, 8)` as requestId at handler start. Include in all error responses and console.error calls for production debugging.
