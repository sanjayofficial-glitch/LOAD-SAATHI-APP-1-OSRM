# Firebase Realtime Database Chat Migration Plan

## Overview
Replace the broken Supabase-based chat with Firebase Realtime Database. Keep Clerk for auth and Supabase for everything else. Only chat messages and conversation metadata move to Firebase.

**Why Firebase**: Solves all 3 chat bugs (broken mark-as-read, fragile realtime, "Unknown" names) with a purpose-built real-time database. Free tier covers moderate usage.

---

## Architecture

```
Clerk Auth → Supabase Edge Function → Firebase Custom Token → Firebase RTDB
                    ↓
        Partner resolution stays in Supabase (requests/shipment_requests tables)
```

- **Firebase RTDB**: Stores messages and conversation metadata
- **Supabase**: Stores trips, shipments, requests, users (everything except chat)
- **Clerk**: Primary auth (generates Firebase Custom Token via Edge Function)

---

## Firebase RTDB Structure

```
/load-saathi/
├── messages/{requestId}/{pushKey}
│   ├── id, sender_id, recipient_id, content
│   ├── request_id, is_read, created_at
│   ├── sender_name, recipient_name (denormalized)
│
├── conversations/{requestId}
│   ├── last_message, last_message_time, last_sender_id
│   ├── participants/{clerkUserId}: { name, user_type }
│   └── unread/{clerkUserId}: number (counter)
```

## Implementation Steps

### Step 1: Create Firebase project + install SDK
- Create project in Google Cloud Console
- Enable Realtime Database
- `npm install firebase`
- Add env vars: `VITE_FIREBASE_API_KEY`, `VITE_FIREBASE_DATABASE_URL`, `VITE_FIREBASE_AUTH_DOMAIN`, `VITE_FIREBASE_PROJECT_ID`

### Step 2: Create Firebase config
**Create**: `src/config/firebase.ts`
- Initialize Firebase app
- Export `db` (Realtime Database instance)
- Export `authenticateWithFirebase(clerkToken)` — calls Edge Function to get Custom Token

### Step 3: Create Firebase Auth Edge Function
**Create**: `supabase/functions/firebase-auth/index.ts`
- Receives Clerk JWT in Authorization header
- Verifies via Supabase auth
- Mints Firebase Custom Token with `uid = Clerk user ID`
- Returns token to client

### Step 4: Rewrite chat utilities
**Replace**: `src/utils/chat.ts`
- `sendMessage()` — writes to `/messages/{requestId}` + updates `/conversations/{requestId}`
- `fetchMessages(requestId)` — reads from `/messages/{requestId}`
- `markMessagesAsRead(requestId, userId)` — flips `is_read` + resets unread counter
- `subscribeToMessages(requestId, callback)` — Firebase `onValue` listener
- `fetchConversations(userId)` — reads `/conversations/` where user is participant
- `subscribeToConversations(userId, callback)` — real-time conversation list updates

### Step 5: Update chat types
**Modify**: `src/types/chat.ts`
- Add `sender_name`, `recipient_name` to Message
- Add `ConversationMeta` interface for Firebase conversation structure

### Step 6: Update Chat.tsx
**Modify**: `src/pages/Chat.tsx`
- Authenticate with Firebase on mount (one-time)
- Partner resolution stays in Supabase (requests/shipment_requests queries)
- Use `fetchMessages` + `subscribeToMessages` from Firebase
- `markMessagesAsRead` now works (Firebase `update()`)

### Step 7: Update ChatList.tsx
**Modify**: `src/pages/ChatList.tsx`
- Replace Supabase query with `fetchConversations` from Firebase
- Add `subscribeToConversations` for real-time updates
- Participant names come from `conversations/` metadata (no more "Unknown")

### Step 8: Firebase security rules
```json
{
  "rules": {
    "messages": {
      "$requestId": {
        ".read": "auth != null && root.child('conversations').child($requestId).child('participants').child(auth.uid).exists()",
        ".write": "auth != null && root.child('conversations').child($requestId).child('participants').child(auth.uid).exists()"
      }
    },
    "conversations": {
      "$requestId": {
        ".read": "auth != null && data.child('participants').child(auth.uid).exists()",
        ".write": "auth != null && data.child('participants').child(auth.uid).exists()"
      }
    }
  }
}
```

### Step 9: Migration script (one-time)
**Create**: `scripts/migrate-chat-to-firebase.ts`
- Reads all messages from Supabase `messages` table
- Writes to Firebase RTDB with conversation metadata
- Run once before deploying new chat code
