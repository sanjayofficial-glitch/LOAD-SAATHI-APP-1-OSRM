# LoadSaathi Dark Mode Refinement Plan

## Overview
The dark mode implementation has significant gaps, especially in charts/graphs. The **monthly spend graph shows blank** in dark mode because recharts defaults to black text/colors on a dark background. Additionally, `TripDetail.tsx` has zero dark mode support, and several components have missing dark variants.

**Root cause of blank graphs**: `CartesianGrid stroke="#f0f0f0"` is invisible on dark backgrounds. XAxis/YAxis labels default to black text (invisible on dark). Tooltip has no background override (white popup on dark card).

---

## Phase 1: Fix Charts (the primary complaint)

### 1A. Shipper Dashboard — Monthly Spending chart
**File**: `src/pages/shipper/Dashboard.tsx` (lines 408-421)

Current broken state:
```
<CartesianGrid strokeDasharray="3 3" stroke="#f0f0f0" />
<XAxis ... tick={{ fontSize: 12 }} />  ← no fill color, defaults to black
<YAxis ... tick={{ fontSize: 12 }} />  ← no fill color, defaults to black
<Tooltip ... contentStyle={{ borderRadius: 12, border: '1px solid #bfdbfe', ... }} />  ← no background
```

Fix:
- `CartesianGrid`: Change `stroke="#f0f0f0"` to `className="stroke-muted"` (maps to CSS variable)
- `XAxis`: Add `tick={{ fontSize: 12, fill: 'hsl(var(--muted-foreground))' }}`
- `YAxis`: Add `tick={{ fontSize: 12, fill: 'hsl(var(--muted-foreground))' }}`
- `Tooltip`: Change `contentStyle` to `{ background: 'hsl(var(--card))', border: '1px solid hsl(var(--border))', borderRadius: 12, fontSize: 12 }`
